# Create and connect properties

Properties and variants allow you to create versatile components that adapt to different needs, eliminating the need to build multiple separate versions of the same element. They define what is changeable on an instance. Instead of making 50 different components for every possible situation (e.g., Button-Red-Small-WithIcon, Button-Blue-Large-NoIcon), you make one component and give it properties.


### Properties vs Variants
Properties give you granular control over an individual instance. You can connect and modify any specific attribute—such as text content, colors, or sizes—directly at the instance level while keeping it linked to the Main Component.
Whereas variants allow you to quickly swap between pre-made configurations or states of a component. For example, a single Button component can include variants like button-default, button-icon, or button-disabled, letting you switch its entire appearance instantly.


### Why Properties Matter 
Properties transform a static element into a functional, dynamic UI component. They provide the flexibility to control how an element looks and behaves based on specific logic, ensuring your design is both adaptable and interactive.
